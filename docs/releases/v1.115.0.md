# Release: v1.115.0: Account API Integrations

## Major Changes

This release completes the API integration feature for accounts, enabling users to track and manage their API integrations within the personal CRM. The integration system has been fully implemented and added to the navigation menu for easy access.

Additionally, a context-aware feature for Bible reading has been restored - when the context is set to "Family", the Bible reading functionality becomes available again.

## Fixes

Several deployment-related issues were resolved during this release cycle:
- Fixed package lock synchronization issues
- Resolved deployment problems related to Node.js version compatibility (tested with Node 20 and Node 22)
- Fixed YAML dependency issue (pinned to version 2.8.1)

## Details

### Account API Integration Feature

The account API integration feature has been fully implemented, allowing users to:
- Manage API integrations associated with their accounts
- Access integration management through the navigation menu
- Track integration-related information within the account context

This feature enhances the CRM's capability to manage technical relationships and API partnerships alongside traditional account management.

### Context-Aware Bible Reading

Bible reading functionality has been restored with context awareness:
- When the user's context is set to "Family", Bible reading features become available
- This provides better separation of personal/family content from work-related content
- Enhances the multi-context support that separates different areas of life within the CRM

### Smaller Changes

- Multiple pull request merges for the account integration finalization (#323, #324, #325, #326)
- Package dependency synchronization and updates
- Deployment configuration improvements for Node.js compatibility
